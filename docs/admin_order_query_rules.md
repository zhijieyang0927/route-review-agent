# Admin 订单查询规则

本文记录饭团后台 `运营中心 -> 订单管理 -> 订单查询列表` 的检索口径，用于后续自动检索订单日志。

## 固定入口

- 后台入口：`https://adm.fantuan.ca/#/delivery/start`
- 常用页面：`https://adm.fantuan.ca/#/customer/orderManage/orderManageList`
- 功能路径：`运营中心 -> 订单管理 -> 订单查询列表`
- 默认订单类型：`外卖/自提/跑腿/团送`
- 默认国家：`US`
- `公众号` 默认不操作。

## 搜索类型

### 单号

- 美国区域单号一定以 `US` 开头。
- 后续如果负责其他国家/区域，单号前缀和格式由用户另行更新。
- `US` 后通常是 9-11 位数字。
- 总长度大致是 11-13 个字符。
- 单号是唯一值，不会重复。
- 示例形态：`US0590379572`。

### 取餐号

- 取餐号是 5 位纯数字。
- 取餐号会在一定时间周期内重复使用，不是全局唯一。
- 搜索结果列表最左侧会显示取餐号，展示时会带 `#` 前缀。
- 判断结果是否匹配时，只要 `#` 后面的 5 位数字与搜索输入一致，即认为取餐号匹配。

### 订单ID

- 暂时不用。
- 只有当用户明确说明输入号码是 `订单ID` 时才使用该搜索类型。

### 电话

- 电话是下单客人的电话号码，不是商家电话。
- 输入前需要去除所有格式符号，只保留数字。
- 示例：`(860)617-4298` 应归一化为 `8606174298`。
- 后续如果用户说“搜索订单”并提供电话号码，应默认按客人电话搜索。

### 地址

- 地址指客人家的地址。
- 通常没有特殊固定格式。

### 餐厅名称

- 餐厅名称指订单里的商家名字。
- 后台商家全名通常较长，可能包含宣传标语或活动信息，例如 `0.99大鸡排`、`云南正宗口味`、`9.99午饭套餐`。
- 连锁店或多门店商家名称可能包含区域或街道区分，例如 `法拉盛`、`86街`、`北方大道店`。
- 后续用户通常会提供商家的主要名称，而不一定提供完整后台名称。
- 搜索结果里可能出现同名但不同区域/街道的商家。
- 结果栏的 `门店` 会展示 `餐厅ID + 餐厅完整名称` 的组合。
- 当不确定是否为同一家店时，优先通过 `餐厅ID` 确认是否一致。

### 餐厅ID

- 餐厅ID只包含数字。
- 字段长度随机，可能为 1-7 位。
- 餐厅ID完全独立且不会重复。
- 每个餐厅ID对应一家单独的后台商家店铺。

### 批量送团ID

- 批量送团ID属性类似单号。
- 字段比普通单号稍短。
- 完全独立，不会重复。
- 短期内不会用到。
- 只有当用户明确说明输入的是 `批量送团ID` 时才使用该搜索类型。

## 展开搜索栏

点击 `展开搜索栏` 后会显示更多筛选项。

### 下单日期

- `下单日期` 是以客人下单时间为基准的时间过滤条件。
- 主要影响不具有全局唯一性、可能重复的搜索字段，例如：
  - 取餐号
  - 电话
  - 地址
  - 餐厅名称
  - 餐厅ID
- 对具有唯一性的字段，通常不应受下单日期影响返回多条结果：
  - 单号
  - 订单ID
  - 批量送团ID
- 如果使用 `单号`、`订单ID`、`批量送团ID` 检索时返回多于一条结果，需要正常展示结果，同时提醒：`数据返回异常`。
- 后续用户提供时间段时，默认按 `下单日期` 过滤，除非用户明确说使用 `完成日期`。

### 完成日期

- `完成日期` 是以客人订单完成配送的时间为基准的时间过滤条件。
- 仅当用户明确要求按完成日期检索时使用。

### 用户ID

- 用户ID与餐厅ID一样具有独立性。
- 每个用户ID只对应一个单独用户账号。
- 用户ID以字母 `T` 开头，后面跟一段数字。

### 快捷选择

- 快捷选择有两个主要选项：
  - 外卖订单
  - 团送订单
- 搜索外卖订单时，必须检查并确认 `外卖订单` 已勾选，否则结果栏可能返回不正确。
- 默认所有查询都按 `外卖订单` 勾选处理。
- `团送订单` 默认关闭。
- 只有当用户明确说查询团送订单时，才勾选 `团送订单`。

### 订单类型

- 常用订单类型：
  - 自提
  - 外卖
  - 次日达
  - 帮我买
  - 取送件
  - 团送
- 平时后台系统默认这些类型为勾选状态。
- 暂不使用：
  - 会员
  - 购买券包
  - 钱包充值
  - 礼品卡
  - 校园餐
- 如后续需要使用上述暂不使用类型，用户会另行说明操作流程和过滤规则。

### 订单状态

- `待接单`：
  - 指客人已经下单、订单已经生成，等待商家接单。
  - 对应结果栏客服状态中的 `已接单`，即等待商家接单状态。
- `已接单`：
  - 指商家已确认接单并开始准备餐品。
  - 对应结果栏客服状态中的 `已推送`。
  - 该状态下订单还会被 `POS状态` 细分为 `已接单` 和 `已推送`：
    - POS状态 `已接单`：商家已经接单。
    - POS状态 `已推送`：商家已收到订单推送，但尚未接单。
- `已指派`：
  - 指订单已经指派给一位送餐员配送。
  - 对应结果栏客服状态里带有骑手编号的 `已推送`。
- `已取餐`：
  - 指送餐员已经取餐。
  - 对应结果栏客服状态中的 `正在配送`。
- `已完成`：
  - 指配送完成的订单。
  - 搜索所有订单时默认应勾选，否则会导致部分取餐号订单在日期范围内返回不全。
  - 对应结果栏客服状态中的 `已完成`。
- `已关闭`：
  - 指不论什么原因被关闭的订单。
  - 搜索所有订单时默认应勾选，否则会导致部分取餐号订单在日期范围内返回不全。
  - 对应结果栏客服状态中的 `已关闭`。
- `待截单`、`已截单`：
  - 专门对应团送订单。
  - 通常不用。
  - 当使用 `批量送团ID` 或用户要求检索团送订单时，可以默认勾选。
  - 非团送查询时保持默认关闭。

### POS状态

- 暂不使用。
- 保持系统默认。

### 支付方式

- 暂不使用。
- 保持系统默认。

### 标记状态

- 标记状态选项对应结果栏里订单号前面的标识。
- 标记状态共有 22 个选项。
- `正常订单` 是未加星状态。
- 其他标识含义可在点击标记图标后出现的弹窗中查看。

### 暂不同步/暂不用字段

以下字段暂时不用，不需要同步具体含义：

- 晚单时间
- IP地址
- 服务语言
- 关闭原因
- 用户语言
- 设备类型
- 渠道来源
- 特殊订单
- 用户画像

### 重置

- `重置` 按钮会恢复上方所有过滤信息和选项到系统默认设置。
- `重置` 会清除搜索输入框内容。
- `重置` 后，下方结果栏会按订单生成时间从新到旧展示当前所有订单结果。
- 本轮学习结束后，需要点击一次 `重置` 并记录系统默认过滤条件。

#### 已观察到的系统默认状态

点击 `重置` 后，当前观察到的系统默认状态如下：

- 搜索类型：`单号`
- 搜索输入框：空
- 下单日期：空
- 完成日期：空
- 用户ID：空
- 快捷选择：
  - `外卖订单`：已勾选
  - `团送订单`：未勾选
- 订单类型：
  - `自提`：已勾选
  - `外卖`：已勾选
  - `次日达`：已勾选
  - `帮我买`：已勾选
  - `取送件`：已勾选
  - `团送`：已勾选
  - `会员`、`购买券包`、`钱包充值`、`礼品卡`、`校园餐`：未勾选
- 订单状态：
  - `待接单`：已勾选
  - `已接单`：已勾选
  - `已指派`：已勾选
  - `已取餐`：已勾选
  - `正在配送`：已勾选
  - `已完成`：未勾选
  - `已关闭`：未勾选
  - `待截单`：未勾选
  - `已截单`：未勾选
- POS状态：系统默认未观察到勾选项。
- 支付方式：系统默认未观察到勾选项。
- 标记状态：系统默认未观察到勾选项。

#### 复盘查询建议默认

- 对非唯一字段做完整复盘查询时，尤其是 `取餐号`、`电话`、`地址`、`餐厅名称`、`餐厅ID`，应根据用户提供的时间范围设置 `下单日期`。
- 搜索“所有相关订单”时，除系统默认订单状态外，应额外勾选：
  - `已完成`
  - `已关闭`
- 否则在日期范围内可能漏掉已完成或已关闭的订单。
